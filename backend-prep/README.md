## Key Findings

### The frontend is fully self-contained
Every data path terminates at `localStorage` or hardcoded mock data. There are **zero** real HTTP calls to a backend (except optional Cloudinary image uploads).

### Six localStorage keys hold all state
| Key | Content |
|-----|---------|
| `frill_token` | Base64-encoded "JWT" (not a real JWT) |
| `frill_users` | Array of user accounts (plaintext passwords) |
| `frill_orders` | Array of order records |
| `frill_designs_v2` | Array of saved designs |
| `persist:frill-cart` | Redux-persisted cart state |

### 21 API endpoints are needed
- **13 P0** (must-have for MVP): auth, products, orders, media upload
- **6 P1** (needed for full feature): password reset, designs, profile
- **2 P2** (nice-to-have): admin stats

### 3 bugs found
1. Order status case mismatch between admin pages
2. Checkout silently succeeds on API error
3. Password reset page does nothing

### Significant dead code
- Entire Fabric.js studio (`features/design-studio/`) — 684 lines, unused
- Legacy `productsSlice.js` + `useProducts.js` — superseded by RTK Query
- Duplicate `useScrollLock.js` / `useScrolled.js` hooks

- Line numbers cited for every factual claim
